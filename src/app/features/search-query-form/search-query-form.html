Search Query Form

Config-driven filters and optional sorting that emit the same paginated search request shape used by the backend.

Empty by Default

Optional properties stay hidden until a user adds them. With no required config, the emitted request has no filters. Name and Created At allow custom In values and are also available in the sort controls. Sorting supports up to two unique properties, with Name ascending restored as the default.

Required and Multi-Value Filters

The Name filter demonstrates custom string values with the In operator alongside option-only, mixed option and custom, and other typed multi-value filters in one backend-compatible request. Enum options use display labels with backend string values: users see Engineering and Operations while state emits ENG and OPS. The multi-select creates compact priority chips; each chip can toggle direction or be removed. Sorts are emitted in chip order, limited to three unique properties, and reset to Name ascending then Created At descending when Clear filters is selected. Reset defaults restores the required Tenant plus the default-visible Status and Created At filters with their configured values.
