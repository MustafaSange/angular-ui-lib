Lazy children sources

Load children from a Promise, the first Observable emission, or a Signal that changes from null to a child array.

When loading runs

  1. Expanding an unloaded node calls its loader once.
  2. The node shows loading until its child array arrives.
  3. Successful children are cached, so reopening does not call the loader again.
  4. A failed load waits for the user to select Retry.

Promise

Resolve to child nodes or reject to show a configurable error and retry action. The Failed jobs node intentionally rejects once.

Observable

The first emitted child array is cached and the subscription ends.

Signal

Return null while loading, then set the Signal to the child array.

Keyboard, disabled, and RTL

Arrow keys follow logical direction. Type characters to find visible nodes by label. Disabled nodes remain visible but are skipped by focus and type-ahead.
