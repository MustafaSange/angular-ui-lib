# Feature 034: Utilities Showcase

## Goal

Create a dedicated showcase page for the global utility classes under `src/styles/utilities/`.

The page should help users understand where utility classes are useful, see a realistic UI example,
and inspect the complete class reference for each utility group.

## Context

The utility classes themselves are documented in `context/002-utility-classes.md`.

This feature documents the Angular showcase page at:

`src/app/features/utilities/`

The showcase is linked from Home and routed at `/utilities`.

## Page Structure

The Utilities page includes:

- a back link to Home
- a page heading and short description
- a responsive grid of utility group cards
- an end-side drawer for the selected utility group

Utility groups:

- Display
- Flex
- Gaps
- Spacing
- Typography
- Colors
- Borders
- Shadows
- Sizing
- Position
- States

Each card shows:

- category
- utility group name
- short summary
- practical benefit
- a small neutral preview graphic

Cards should not show class chips.

## Drawer Content

Opening a utility card shows an `ms-drawer` with:

- utility category and title
- purpose summary
- practical benefit
- short guidance copy
- a realistic example using that utility group
- classes used in the realistic example
- Use when / Avoid when guidance
- all available classes for the group
- a copyable `ShowcaseCode` snippet

The drawer uses:

```html
<ms-drawer placement="end" aria-labelledby="utility-detail-title">
```

The drawer may set `--drawer-inline-size` to create a wider detail surface.

## Styling

Use existing tokens for colors, spacing, radius, borders, shadows, and focus states.

Keep the UI neutral and readable:

- avoid strong selected card colors
- use restrained borders and surfaces
- keep examples realistic rather than abstract
- avoid class chips on the card grid
- keep code chips padded enough to read comfortably

Reusable layout should use logical CSS properties.

The shared drawer supports:

```scss
--drawer-inline-size
```

This custom property lets the Utilities showcase use a wider drawer while preserving the default
drawer width for other consumers.

## Showcase Code

Use `ShowcaseCode` from:

`src/app/shared/ui-lib/components/showcase-code`

Snippets should be hand-authored in `utilities.ts` and match the realistic example closely enough
that copied code demonstrates the same utility concept.

## Accessibility

- Utility cards are native buttons.
- The active card uses `aria-pressed`.
- The drawer uses `aria-labelledby`.
- The drawer close button has an accessible name.
- Decorative previews use `aria-hidden="true"`.
- Drawer focus and keyboard behavior come from `ms-drawer`.

## Routing

Add the route:

```ts
{
  path: 'utilities',
  loadComponent: () => import('./features/utilities/utilities').then((m) => m.Utilities),
}
```

Add a Home showcase card linking to `/utilities`.

## Acceptance Criteria

- Visiting `/utilities` shows the Utilities showcase.
- Every utility group opens a drawer with a realistic example.
- The drawer lists classes used in the example.
- The drawer includes Use when / Avoid when guidance.
- The drawer includes a complete class reference for the selected utility group.
- The page is linked from Home.
- `npm run build` passes, aside from any pre-existing bundle budget warning.
